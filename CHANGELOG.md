# 更新日志

## v1.8.0（2026-07-24）

### Fable 审计落地
- **环境自检卡**：只读检查酒馆上下文、完整 MVU API、故事神谕 AUTO、MVU 额外解析运行状态、活世界注入通道、上轮注入落地和模型连接；不会为了“检测”擅自修改设置。
- **设置页重新分层**：状态总览与时间线常开，变量、硬合同、活世界、论坛和进阶设置默认折叠；完整事件账本只保留在悬浮面板，不再重复两份。
- **长任务阶段与软取消**：变量诊断明确显示读取 MVU、稳定等待、构建上下文、第 x/3 次模型分析、本地校验、写入回读；停止后使用 epoch 守卫丢弃迟到结果，宿主明确支持时同时发送 AbortSignal。
- **完整提示词透明**：显示最后一次真实提示词的任务、字符数和分段规模，可复制或下载完整内容；脱敏诊断包不包含正文、世界书、变量原文、提示词或密钥。
- **按聊天模型调用统计**：分别记录变量、活世界、论坛、失败和 429 次数并跨刷新保存，公益站用户可以直接核对真实调用量。

### 手机前端与可访问性
- 事件卡摘要只保留阶段与紧迫度，详细字段分为「当前 / 因果 / 传播与收束」，默认只展开当前并隐藏空值；事件进度条补齐无障碍属性。
- 论坛增加相对页次、新内容提示、加载骨架和评论者首字头像；悬浮球长按可归位，贴边状态扩大隐形触控区。
- 悬浮面板和论坛补齐焦点进入、Esc/关闭后焦点归还、焦点循环及 `aria-modal`；手机可点元素统一触控下限。
- 状态色、悬浮球渐变与背景改为酒馆主题语义色，补齐浅色主题回退；危险清空操作改为主题弹窗并显示将删除的数量。

### 通用性、诊断与工程质量
- 正文硬合同可识别 `<content>` 之外的常见叙事包装（如 `<gametxt>`、`<storytext>`、`<正文>`）；未知包装会明确安全降级，不把状态栏误算成正文。
- 活世界注入加入哨兵和最终提示词只读探针，可判断「已登记但未落入最终 prompt」；公开 API 升为带兼容检查的 v1，并补充自检、诊断、调用统计与注入状态接口。
- 修复 `npm test`/CI 实际未执行浏览器测试的问题：锁定 Playwright，CI 安装 Chromium，缺浏览器依赖时 CI 直接失败而不是静默跳过。
- 保持既有成本与安全语义：不新增全局 token 限制；只有变量分析结果损坏时最多三次尝试；正文修正版仍复用同一次变量调用并经过玩家行动守卫、写前恢复、写后回读和撤销保护。

### 版本
- manifest.json / index.js / package.json / package-lock.json 统一升级至 1.8.0。

## v1.7.2（2026-07-24）

### 新增
- **通知级别设置**：设置页新增「通知级别」下拉（全部 / 只弹警告与失败 / 静默），成功类 toast 不再强制打扰；所有状态仍完整记入操作时间线。
- **最近操作时间线**：悬浮面板「工具」页新增可折叠的操作时间线，保留最近 30 条带时间戳的状态记录（变量 / 硬合同 / 世界 / 论坛），历史状态不再被最新一句覆盖。
- **工程基建**：新增 `package.json`（`npm test` 一键回归）、MIT `LICENSE`、GitHub Actions CI（push/PR 自动跑全部测试）、版本一致性测试（manifest / index.js / package.json 三处版本必须一致）。

### 修复与增强
- **悬浮球状态光效改为 kind 聚合**：不再对状态文案做正则匹配，改用各状态显式传入的 kind；今后调整文案不会再让球体光效失灵（潜在 bug 修复）。
- **零改动修正版拦截**：模型偶尔原样返回正文却声称已修正，现在会在本地守卫阶段直接拒绝，不再生成无意义的新 swipe。
- **变量诊断"成稿纪律"提示**：system 提示新增一条指令，要求推理模型内部一遍推理完成后一次性输出全部区块，减少 32K 预算被反复重想耗尽导致的截断。

### 版本
- manifest.json / index.js / package.json 统一升级至 1.7.2。
