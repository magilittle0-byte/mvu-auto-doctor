# MVU 自动医生（通用）

这是一个独立的 SillyTavern / TauriTavern 扩展。它包含通用 MVU 审计修复和可选的平行支线连续性账本，不会修改角色卡、SP·数据库、预设、缝合怪或 Story Oracle 文件。

## 安装

在 TauriTavern 的扩展安装页面选择“从 Git 仓库安装”，粘贴：

```text
https://github.com/magilittle0-byte/mvu-auto-doctor
```

安装后刷新一次页面。后续版本可直接使用扩展管理里的“更新”功能，不需要重新下载压缩包。

每次收到新的 AI 回复后，扩展会：

1. 动态读取当前角色卡暴露的 MVU/Zod Schema。
2. 动态读取当前启用世界书中的 `[mvu_update]` 规则。
3. 等待本轮普通 MVU 更新稳定，避免在变量仍写入时抢跑。
4. 对比上一 AI 楼层状态、当前 `stat_data`、本轮正文和原更新区块。
5. 让模型只生成针对当前状态的纠错/补漏补丁。
6. 在内存副本上校验路径、操作、MVU 解析和 Zod 结果。
7. 全部通过后才原子写入，并在写入后回读验证。
8. 把纠错块持久附加在当前 swipe 中；刷新或重进聊天后，原错误块与纠错块会按顺序重放，不会把旧错误复活。
9. 刷新消息与 `<StatusPlaceHolderImpl/>`，让正文状态栏立即重建。

v1.2.0 开始，每次任务还会绑定聊天 ID、稳定消息 ID、楼层、swipe、正文指纹和生成世代，并在每个异步步骤后复核。数据库预热/算 Token 的 `dryRun` 会被忽略；普通生成、swipe、regenerate 与 continue 分开记录。真正写变量时使用精确楼层而不是笼统的 `latest`，最大限度避免切聊天、重抽或连续生成时落错位置。

模型连接顺序：

- 默认优先使用 Story Oracle 已配置的连接（稳定 Hook API v1）。
- Story Oracle 未安装、未配置或调用失败时，自动使用酒馆当前主连接的 `generateRaw`。

为避免两个自动程序同时写变量，扩展默认会关闭 Story Oracle 的 `AUTO` 诊断开关；Story Oracle 的手动诊断不受影响。

## 必须这样设置

- 关闭 MVU 的“额外 AI 解析/额外模型更新变量”。本扩展跟随角色卡原本的 `<UpdateVariable>` 更新完成后再审计，不需要第二个 MVU 解析模型。
- 只保留一个自动修复器：开启“MVU 自动医生”，关闭 Story Oracle 的 `AUTO` 诊断。Story Oracle 的手动诊断、问剧情、校正回复、世界书模式仍可正常使用。
- SP·数据库继续使用作者原版填表提示词。自动医生不读写数据库表，也不接管数据库回忆。
- 同一角色只启用一套当前版本的 MVU 更新规则。若作者卡已内嵌世界书，不要再同时启用一套重复旧世界书。

“关闭 MVU 额外 AI 解析”只针对 MVU 自带的第二次变量解析。支线连续性开启时，会在主回复落地后使用插件连接整理一份独立的因果账本，但它不写 MVU，也不代替角色卡原本的变量更新。

## 平行支线连续性

该功能不会在正文末尾再追加一套平行事件，而是把状态保存在当前聊天的插件元数据中，并在下一回合以短账本注入：

- 识别预设的 `<parallel_event_record>`、`<current_event>`，以及缝合怪的 `<dm_story>`、`<npc_track>`。
- 给支线保留稳定 ID、阶段、触发条件、下一拍、知情层级与来源回复。
- 每回合最多让一条支线产生可观察变化；已有事件优先，避免反复生成相似伏笔。
- hidden / rumor / observed 分层，禁止让不知情角色突然全知。
- 只推动 NPC、势力、敌方、环境、约定与离场角色；禁止替玩家角色行动、回答、移动、消费资源或追加检定。
- 检测到预设或缝合怪时采用桥接模式，由原系统保留剧情提案权；插件只做接续、去重和回收。
- 没有检测到其他推进器且选择“始终开启”时，才允许从已出场人物和既有事实中低频建立支线，不凭空开新世界观。

模式说明：

- `自动兼容（推荐）`：只有检测到预设/缝合怪标记或当前聊天已有支线时运行。
- `始终开启`：没有结构化标记时也整理正文中的明确未决因果。
- `关闭`：撤下注入，不运行支线整理；已有账本仍保留，除非手动清空。

重抽或重新生成时，正文生成阶段使用该楼回复产生前的支线存档点；新 swipe 落地后再从存档点重算，不让被放弃的旧 swipe 污染新剧情。

### 手机端查看支线账本

在“扩展设置 → MVU 自动医生（通用）→ 平行支线连续性”中可以直接查看当前聊天的支线账本：

- 顶部显示未结数、已收束数、账本轮次、更新时间和当前来源模式。
- 未结支线按“已显现 → 推进中 → 已埋设 → 搁置”及紧迫度排序；第一条默认展开。
- 点击每条支线可查看当前进展、下一自然接口、触发条件、涉及人物/势力、地点、知情范围和最近来源楼层。
- 已收束支线放在单独的折叠区，不与活动支线混在一起。
- “刷新显示”只重读当前聊天元数据，不调用模型；“立即整理支线”才会调用模型更新账本。
- 该区域是玩家审计视图，可能显示角色尚不知情的幕后信息；插件不会因此把隐藏知识泄露给正文角色。

切换聊天、收到新回复、整理完成、重抽或清空账本后，显示会自动刷新。选择“关闭”只停止后续整理，旧账本仍可查看。

扩展设置里提供：

- 自动开关
- 模型连接优先级
- 双写保护
- 回复后等待时间
- 手动检查最新回复
- 跨刷新保存、并经过变量指纹保护的撤销记录
- 平行支线的自动兼容/始终开启/关闭模式
- 手动整理支线与清空当前聊天账本
- 手机端可展开的支线账本明细与已收束归档

插件不包含任何角色卡专属字段、路径或枚举。角色卡更新或切换到其他 MVU 卡时，会重新读取新卡的规则和 Schema。

聊天元数据只保存有限条修复快照、支线账本和来源指纹，不保存 API 密钥或完整模型提示词。单条 MVU 快照超过安全体积时不会写入聊天，此时该条修复仍然有效，但无法跨刷新撤销。
